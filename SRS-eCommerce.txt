Requirments Elicitation and Analysis
-------------------------------
SRS (Software Reqiremnets Specification Document)
-------------------------------
1. The Login form will ask for a USERNAME and PASSWORD combination which
will be verified before the user is allowed to enter the website.
2. The Sign Up form will contain EMAIL, USERNAME and PASSWORD fields, all
of which are compulsory to enter.
3. The user should be able to View the products as a list. It should display
the product PICTURE, NAME, CATEGORY and SMALL DESCRIPTION. The user should
be able to click on a product to view full details.
4. The page displays ALL THE DETAILS OF A PRODUCT.
Details : PICTURE, NAME, CATEGORY, SUBCATEGROY, PRICE, UNITS, DESCRIPTION
5. The user should be allowed to add a product to the cart from the Details
Page.
6. The user can View the all the products that they've added along with the
total price of the entire order.

(i) Functional
(ii) Non-functional
(iii) Stakeholder
7. The order will be processed after the user makes payment successfully.

Ideal : The signup form will contain a USER BIRTHDATE FIELD which will be
in the format DD-MM-YYYY and it is compulsory for the user to fill.
